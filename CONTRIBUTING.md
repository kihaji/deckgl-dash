# CONTRIBUTING

This project was generated by the [dash-component-boilerplate](https://github.com/plotly/dash-component-boilerplate) it contains the minimal set of code required to create your own custom Dash component.

## Dependencies and reproducible builds

This package **ships its built JS bundle** (`deckgl_dash/deckgl_dash.min.js`), so the published artifact depends on exactly which dependency versions were installed when the bundle was built. To keep that deterministic:

- Runtime deps in `package.json` use **tilde (`~`) ranges** — patch updates are automatic, minor/major bumps are deliberate, reviewed changes.
- **Release builds must install with `npm ci`** (never `npm install`), so the bundle is built from `package-lock.json` exactly. The hatchling build hook (`hatch_build.py`) does this automatically inside `uv build`.
- When you intend to update dependencies, change `package.json`, run `npm install` to re-resolve the lockfile, rebuild, test, and commit both files together.
- Python deps are managed with **uv** (PEP 621 metadata): run `uv lock` after editing `pyproject.toml` and commit `uv.lock`. `uv sync` creates/updates `.venv` with the dev and docs groups.

## Dev environment quick start

```bash
uv sync          # Python env (.venv) with dev + docs groups; builds the JS bundle if missing
npm ci           # JS deps (Node 20+, see .nvmrc)
make build       # rebuild the JS bundle after editing src/lib
make quality     # ruff + pyright
uv run pytest    # Python tests
npm test         # JS tests
uv build         # wheel + sdist — the wheel builds the JS bundle itself (self-building)
```

The built bundle (`deckgl_dash/deckgl_dash.min.js`) is **not** git-tracked: wheels build it via the hatch hook, and `uv sync` builds it on first checkout.
